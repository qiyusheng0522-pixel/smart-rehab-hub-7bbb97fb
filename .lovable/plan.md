# 实施计划

涉及四个角色端的较大改造，预计修改 `DoctorApp.tsx`、`TherapistApp.tsx`、`NurseApp.tsx`、`PatientsModule.tsx`，并新增若干共享 sheet。

## 1. 移除"退回重评"
全局搜索 `退回重评` / `reEval` / `rebid` 相关按钮、状态、跳转，统一删除按钮、移除相关 sheet 和状态机分支。

## 2. 首次评估改为三段式 Tab
在医师端、治疗师端、护士端的首次评估详情页：
- 顶部保留患者基本信息卡
- 下方加 Tabs：**临床评估 / 康复评估 / 康复目标**
  - **临床评估**：生命体征（BP/HR/SpO₂/T）、生化结果（血常规、肝肾功、电解质）、既往病史、用药史
  - **康复评估**：分方向折叠（心肺方向 / 神经方向 / 骨科方向），每个方向下展示专科评估 + 特殊评估（如 CPET、平衡测试）；评估量表沿用现有展示样式
  - **康复目标**：迁移现有"康复目标"模块，含 AI 生成 + 编辑/删除 + 更新目标
- 每条评估结论统一带 **角色标签 + 评估时间**（医师 / 治疗师 / 护士）
- 内容采用 一级标题 → 二级标题 → 条目 三级层次

## 3. 护士端首次评估补充
新增护理首次评估字段：
- 一般情况（意识、皮肤、压疮风险 Braden、跌倒风险 Morse、VTE 风险 Caprini）
- 管路评估（导尿管、PICC、引流）
- 自理能力 ADL（Barthel）
- 心理与疼痛评估（NRS、HAMD 简版）
- 营养评估（NRS-2002）
完成后写入第 2 节同一三段式视图的"临床评估"+"康复评估"中，护士结论与时间一并展示。

## 4. 康复目标排版优化
- 改成卡片列表：序号 ① ② ③ ... + 目标标题 + 时间维度（短期/中期/长期）badge + 衡量指标 + 编辑/删除
- 多目标场景明确编号
- 顶部 1 行操作条：更新目标 / 新增

## 5. AI 团队会议（分歧自动触发）
新增 `TeamMeetingSheet`：
- 入口：当多角色评估结论分歧度高时（mock 触发条件），首页/详情顶部出现"AI 已发起团队会议"提示卡
- 内容：分歧条目列表（条目 + 各角色观点）
- 各角色发言区（mock 多轮讨论气泡）
- 底部"生成会议结论" → 弹出可勾选同步范围（康复评估 / 康复方案 / 康复医嘱）→ 确认同步
医师端、治疗师端、护士端共用入口

## 6. AI 自动排班
在医师端或治疗师端"康复方案/排课"区域新增 `AutoScheduleSheet`：
- 输入：本周需排课患者列表 + 设备列表 + 治疗师列表
- 模式：排课模式（上午 4 段 / 下午 4 段，每段约 45min）
- 输出：网格视图（行=时段，列=治疗师/设备），自动避开冲突，标注利用率
- 操作：一键 AI 排班 / 手动微调 / 确认发布

## 7. 护士端 AI 自动随访增强
扩展现有 `FollowUpSheet`：
- 加"自动随访"开关，AI 主动外呼/IM 与患者沟通
- 随访问答内容覆盖三大块：
  1. **危险因素控制**（脑损伤：血压 / 血脂 / 血糖 / 体重；其他：相应指标）
  2. **营养处方推送**（基于诊断推送）
  3. **康复评估 & 居家治疗指导**
- AI 结论 → 三选一建议：省人民康复科复诊 / 社区维护 / 居家康复
- 列表展示最终结论 + 触发原因
